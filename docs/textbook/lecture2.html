Chapter 2-a

クラスの基礎

データ（プロパティ）と処理（メソッド）を1つにまとめた「設計図」。

今回おさえてほしい点は2つ。1つ目は、クラスはプロパティ（変数）とメソッド（関数）をセットで持った構造体だということ。2つ目は、クラスは設計図として機能するということです。

これまでは、User 型でデータを定義し、それを使う introduce 関数を別に書いていました。つまりデータと処理がバラバラに宣言されていたわけです。

ただ、この introduce は User 型とセットで使うことが前提の関数です。それなら、データとそれを使う処理をセットで表現できたほうが扱いやすい。それを実現するのがクラスです。

クラス名は大文字始まり（Person、PersonProfile のように単語の区切りも大文字）。クラスが持つデータをプロパティと呼び、const / let はつけません。処理はメソッドと呼び、こちらは function をつけずに書きます。

設計図とインスタンス

コードを見ると、name や age に具体的な値は入っていません。クラス自体は具体的な値を持たない──だから設計図なんです。実際に使うときは new Person("Alice", 20) のように値を入れます。こうしてできた、具体的な値の入ったものをインスタンスと言います。

メソッドの中に出てくる this は、そのインスタンス自身を指します。だから alice というインスタンスでは this.name が "Alice" になります。そして、new したときに走る初期化処理が constructor。受け取った引数を this.name などに代入しています。

POINT
  • クラス＝プロパティ（データ）＋メソッド（処理）のワンセット。
  • クラスは設計図。new で具体値を入れた実体がインスタンス。
  • this はそのインスタンス自身。constructor は new 時の初期化処理。
Chapter 2-b

アクセス修飾子

プロパティを「クラスの外から触れるか」を制御する。

プロパティは、メソッドと同じように alice.name の形で外から直接アクセスできてしまいます。でも、クラスのプロパティは極力、直接アクセスさせないのが一般的です。

たとえば password というプロパティを足すと、alice.password で外から丸見えになります。これは良くないですよね。パスワードのような秘密の値は、外部からアクセスできないように制限をかけたい。そこで使うのが private です。private をつけると、クラスの外からはアクセスできなくなります。

注意
npm run dev（開発中）では private でも値が見えてしまいます。ですが npm run build するとエラーで止まります。VS Code 上でもエラー表示されるので、そこで確認できます。

private なプロパティは、changePassword のような public メソッド経由で操作します。もう1つ readonly。birthYear（誕生年）のように、一度決めたら変わらない値は readonly にします。こうすると、クラスの中からでも上書きできず、設定できるのは constructor の中だけになります。誕生年が後から書き換わる事故を防げます。

POINT
  • 基本は private（または protected）。外からアクセスが必要なときだけ public。
  • 秘密の値は private、初期化後に変わらない値は readonly。
  • プロパティに関する処理をクラス内で完結させると、動きの見通しが立ちやすい。
Chapter 2-c

アクセス修飾子（補足）

修飾子はプロパティだけでなく、メソッドにもつけられる。

前の説明だと「アクセス修飾子はプロパティだけ」と聞こえたかもしれませんが、メソッドにもつけられます。メソッドは外から呼び出すものが多いので public で宣言することが多いですが、クラスの中だけで使いたいメソッドや、今後の継承で使うメソッドには protected がよく使われます。

なお constructor には基本つけません。それ以外のメソッドには、明示的に修飾子を書いておくのがよいでしょう。

POINT
メソッドにも public / private / protected をつけられる。外から呼ぶなら public、内部・継承用なら protected。constructor にはつけない。
演習

演習：クラスを書いてみる

本編で学んだクラス・アクセス修飾子を使って、自分で書いてみましょう。

演習 1 ・ Counter クラス

問題

  • 整数のカウンターを表す Counter クラスを作る。
  • 初期値を コンストラクタで受け取る。カウント値 count は private。
  • メソッド：increment()（+1）／decrement()（−1）／reset()（0 に戻す）／getValue()（現在値を返す）。

演習 2 ・ Book クラス（発展）

問題

  • 本の在庫を管理する Book クラスを作る。
  • title・author は readonly、stock（在庫数）を持つ。
  • borrow(amount)：在庫が足りなければ false、足りれば在庫を減らして true を返す。
  • returnBook(amount)：在庫を増やす。getStatus()：状態を表す文字列を返す。
POINT
状態（count / stock）は private にして直接いじらせず、increment や borrow のようなメソッド経由で変える。「在庫が足りるか」の判定もクラスの中に閉じ込めると、使う側は安心して呼べる。
