import / export — コードをファイルに分ける

コードをすべて1ファイルに書くと長くなりすぎます。適切な大きさで別ファイルに分け、必要なものだけ読み込む。その受け渡しをするのが export と import です。

そもそも import / export とは

TypeScript では main.ts などにコードを書いていきますが、全部を1ファイルに書くとすごく長くなります。そこで、適切なサイズで別のファイルに分けて置き、そこに書いた関数・変数・（この後扱う）クラスを、使いたいファイルから import して呼び出します。公開する側が export、読み込む側が import です。

名前付き export（関数）

足し算をする add 関数を functions.ts に書き、main.ts から使いたいとします。ただファイルを分けて移しただけでは動きません──「その関数がどこで定義されたのか分からない」というエラーになります。

そこで関数の前に export をつけます。これで他のファイルから呼び出せるようになります。読み込む側は { add } のように波かっこで受け取ります。

変数も export できる

export は関数だけのものではありません。たとえば円周率 PI を変数として定義し、同じように export をつければ、他のファイルから { PI } で読み込めます。変数・クラスにも同じように使えます。

default export

もう1つ、default export という書き方があります。掛け算の multiply を export default で公開すると、読み込むときに波かっこなしで受け取れます。

ただし特徴（制限）が1つ。1ファイルにつき default export は1つだけ。2つ書くとエラーになります。

読み込む側では、default export の multiply は波かっこなし、名前付きの add・PI は波かっこ、というように1行にまとめて書けます。

POINT
  • import / export は関数・変数・クラスすべてに使える。
  • 名前付き export は { add } と波かっこで、default export は multiply と波かっこなしで受け取る。
  • default export は1ファイルにつき1つだけ。
  • クラスは「1ファイル＝1クラス」で書くのが基本。だから別ファイルのクラスを使うには export / import が必要になる──それでクラスより先にこの話をした。
