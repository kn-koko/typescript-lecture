継承とは — 共通の性質を親クラスにまとめる

犬も猫も「動物」の一種です。共通する性質を親クラスにまとめ、子クラスがそれを引き継ぐ。この仕組みが継承。同じコードの重複を避けるための、クラスの大事な機能です。

コード系レクチャー（Lecture 3）：この回は説明用に Dog / Animal を使います。これらはこの回だけの例で、リポジトリには含まれません（コードは動画の画面から書き起こし）。実際のリポジトリでは、この後 Character を親とした継承で同じ考え方を使います。

継承ってなに？

継承をざっくり言うと──たとえば「犬」や「猫」というクラスがあるとします。犬や猫はもちろんクラスですが、それらの元になる、より抽象的な「動物」クラスの性質も併せ持っています。このとき「犬クラスは動物クラスを継承している」という言い方をします。言葉だけだと分かりにくいので、具体例で見ていきましょう。

まずは継承なしで書いてみる

Dog クラスを作ります。プロパティは名前（name）だけ。コンストラクタで名前を初期化し、名前を返す getName と、鳴き声を返す makeSound の2つのメソッドを持たせます。

猫好きの人のために Cat クラスも作ります。中身は基本的に同じで、makeSound が返す鳴き声だけ違います（"ニャー!"）。

ここで2つを見比べてみてください。name プロパティの宣言・コンストラクタ・getName は、犬も猫もまったく同じですよね。

重複が問題になる

今は2つだけですが、ここから Bird、Rabbit、Pig …と動物クラスを増やしていくと、この共通処理がどんどん増えていきます。関数のときにも話した通り、同じ処理をあちこちにコピペするのは良くない。記述量が増えるだけでなく、変更したいときに全部を直す必要があり、抜け漏れも出ます。共通の処理は、なるべく1か所にまとめておきたい。

親クラスにまとめる（継承）

そこで、犬・猫・鳥…を包括する概念として Animal（動物）クラスを作り、共通部分（name・コンストラクタ・getName）をそこに持たせます。子クラスからはその共通部分を消し、代わりに extends Animal と書きます。これが継承です。

Dog・Cat の実装がぐっとシンプルになりました。共通処理は Animal にあるので、そこを直せば Dog・Cat の両方に反映されます。

実際に使ってみると、Cat には getName もコンストラクタも書いていないのに、ちゃんと動きます。それらは Animal から継承しているからです。

POINT
  • 継承＝共通の性質を持つクラス（親）を引き継ぐこと。子 extends 親 と書く。
  • 共通のプロパティ・コンストラクタ・メソッドを親に1か所にまとめられる。子はそれを継承しつつ、独自のメソッドを足せる。
  • 親を直せば、継承している子すべてに反映される。重複が消え、変更にも強くなる。
この Dog / Animal は説明用の使い捨て例です。次の Lecture 5 では、リポジトリの Character クラスを親として Warrior / Mage などが継承する、という形で同じ考え方を使います。
