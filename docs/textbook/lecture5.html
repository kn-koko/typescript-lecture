Chapter 5-a

継承・抽象クラスの復習

出発点は、Character（名前・HP・showStatus）を Warrior / Mage が継承した状態。

Character クラスは name と hp というプロパティを持ち、コンストラクタで初期化、showStatus で状態を出力します。これを継承した Warrior と Mage は、name・hp・showStatus に加えて、それぞれ固有の攻撃メソッドを持ちます。ここから4つの要点を順に見ていきます。

1super() — 親のコンストラクタを呼ぶ

Warrior に武器（weapon）を持たせたいとします。子クラスで独自のコンストラクタを書くと、親のコンストラクタが上書きされて実行されなくなり、name や hp の初期化が行われません。かといって子側で this.name = name と書くこともできません（親のプロパティを子で勝手に初期化はできない）。

そこで super(name, hp) と書きます。これは親（Character）のコンストラクタを呼ぶ記述です。継承したクラスで独自コンストラクタを書くときは、この super() をほぼ必ずワンセットで書く、と覚えておきましょう。

2protected — 継承先には許すアクセス制限

name を private にすると、インスタンスからのアクセスは防げますが、継承先の Warrior からもアクセスできなくなりエラーになります。private は「そのクラスの中でだけ使える」という意味で、継承先は“外”扱いだからです。かといって public にすると誰からでも触れてしまう。

その中間が protected。インスタンスからは触れないが、継承関係にある子クラスからは触れる。厳しさの順に private → protected → public です。基本は private、継承先で使いたければ protected、どうしてもインスタンスから必要なときだけ public。

修飾子継承先からインスタンスから
private××
protected○×
public○○

3override — 親のメソッドを上書きする

Warrior の攻撃も Mage の攻撃も「行動」です。これを同じ名前 attack のメソッドとして扱いたい。そこで親 Character に汎用的な attack を置き、子クラスで override して独自の処理に書き換えます。

TypeScript では override を書かなくても上書きされますが、明示的に書いておくほうが分かりやすいです。これで Warrior は「武器で攻撃した」、Mage は「魔法を唱えた」と、それぞれ独自の処理になります。

4abstract — 実装を持たない「契約」

親 Character に汎用的な attack の実装があると、子はそれを継承してしまいます。でも本当は、attack の中身は子クラスごとに必ず独自実装してほしい。そこで親側の実装を消し、「attack というメソッドがある」という情報（契約）だけを持たせます。それが abstract メソッドです。

ただし代償があります。attack の中身が決まっていないので、この Character はインスタンス化できなくなります。設計図の中にさらに未完成の設計図（attack）がある状態で、そのままでは動かせないからです。abstract メソッドを持つクラスは abstract class（抽象クラス）になり、継承先で実装して初めて使えるようになります。

POINT
  • super(...)：子で独自コンストラクタを書くなら、親のコンストラクタを呼ぶ。
  • protected：インスタンスからは×、継承先からは○。private と public の中間。
  • override：親のメソッドを子で上書きして独自処理にする。
  • abstract：実装を持たず「必ず実装せよ」という契約。持つクラスは abstract class になりインスタンス化できない。
ここは本当に難しいところ。一度で分かる必要はなく、演習で書きながら体で覚えていく部分です。
Chapter 5-b

補足：checkout を元に戻す

復習で古いコミットを見るために checkout した後、元に戻す方法。

復習では、Git Graph で「lecture 3 課題」のコミットに checkout（一時的に巻き戻し）して中身を見ました。もし同じ手順を手元でやって、そこから差分を積んでしまった場合の戻し方です。

  1. 積んだ差分がいらなければ、ソース管理の「変更の破棄」（元に戻すマーク）で全部破棄してよい。
  2. 元の状態に戻すには、Git Graph で main を右クリック →「Checkout Branch」。これでメインブランチに戻れる。
POINT
過去のコミットを checkout して見るのは自由。戻すときは「変更の破棄」＋ main ブランチへ Checkout Branch。
演習

演習：継承と抽象クラスを使う

学んだ継承・override・abstract を使って、キャラクターと戦闘を作ります。

1Archer クラス

問題

  • Character を継承した Archer クラスを作る。
  • 矢の数 arrows を持ち、コンストラクタで受け取る（super() を忘れずに）。
  • attack を実装：矢が無ければ「矢がない!」、あれば矢を1本減らして「矢を放った!」。

2戦闘シミュレーション（発展）

問題

  • attack が攻撃する相手（opponent）を引数に取るようにする。
  • Character に takeDamage(damage)（HP を減らす。0 未満なら 0）と isDead()（HP が 0 以下か）を追加。
  • attack の中で opponent.takeDamage(...) を呼び、相手の HP を減らす。
  • main.ts で、敵とのターン制バトルを while ループで回す。
POINT
hp は protected なので、相手の HP を直接は触れない。だから opponent.takeDamage(20) のようにメソッド経由で減らす──これがカプセル化。isDead() で勝敗を判定し、while ループで交互に攻撃させれば、継承・抽象クラス・カプセル化がひとつの戦闘として動き出します。
