進め方への要望と、共通して多かった不安にまとめて回答します。

Q1動画の公開をもっと早く・平日のうちにしてほしい

「土日は時間が取れないので平日に見たい」「公休の関係で公開が遅いと課題が間に合わない」という声をいただきました。確約はできませんが、できるだけ早め（平日のうち）の公開を心がけます。間に合わなかった週は、遠慮なくアンケートや授業内でその旨を伝えてください。

Q2オンデマンドだと、課題で詰まったら先に進めなくなりそう

詰まること自体は織り込み済みなので、詰まったまま止まらずに、その状態のまま共有してください。「どこが分からないか」をうまく言語化できなくても、「動画◯-◯のここで手が止まった」「このエラーが消えない」で十分質問になります。エラー文はそのまま貼ってもらえるのがいちばん助かります。アンケート・授業内のどちらでも構いません。

Q3AIに頼らないと1からコードを書けないのが不安

同じ不安を複数の方が書いてくれました。まず、この不安を感じられているのは学習が進んでいる証拠です（写しているだけの段階では、この不安は出てきません）。そのうえで、おすすめの練習を2つ：

  • 「そらで書く」練習 … 何も見ずに Character クラスとその継承先を書けるか試す。書けなかった箇所が「まだ身についていない箇所」の正確なリストになります。
  • AIの使い方を「書かせる」から「説明させる」に寄せる … コードを生成させる代わりに、「このエラーの意味を説明して」「この1行は何をしている？」と聞く使い方にすると、AIを使うほど理解が深まる方向に働きます。

研修の課題は「自力で書く量を確保する」ための場でもあるので、課題ではできるだけ自分の手で書いてみてください。

Q4自分がどれだけ理解できているのか分からない・何が分からないのか分からない

冒頭の集計のとおり、理解度の自己評価は5〜7割のあたりに全員が固まっています。「全体的にふんわり」は今の標準状態です。そのうえで、理解度を測る道具を2つ用意しています。

  • 小テスト … 無記名なので、成績とは無関係に「自分がどこまで分かっているか」の確認に使ってください。間違えた問題が、そのまま復習リストになります。
  • 「そらで書く」テスト … Q3と同じ方法です。「読めば分かる」と「書ける」の差が、いちばん正直な理解度です。

Q5Git・GitHub が不安（操作の意味が分からない・全容が掴めない）

今回いちばん多かった不安です（7名以上）。3つに分けて答えます。

① まず全体像だけ掴む。世界は2つしかありません——手元（ローカルリポジトリ）と GitHub（リモートリポジトリ）。clone／pull は「GitHub→手元」、push は「手元→GitHub」、ブランチは「手元での作業の枝分かれ」、プルリクエストは「枝分かれを合流させる依頼」。個々のコマンドで迷子になったら、いま自分がこの図のどこにいるかだけ確認してください。

② 復習は教材で。Lecture 4（GitHub — fork / clone / push）とLecture 6（チーム開発 Git — ブランチ・プルリクエスト）に、操作画面つきで一連の流れが載っています。

③ 壊すのを恐れない。「正直言われた通りにやっているだけ」で大丈夫な段階です。自信は試行回数からしか生まれないので、自分のリポジトリでなら何度失敗しても問題ありません。間違えて先生のリポジトリにプルリクエストを出してしまっても、closeすれば済むだけで何も壊れません（これは誰もが一度は通る道です）。

Q6クラスのファイルが増えていくと、どこに何を書いたか分からなくなる

ファイルが増えてきたら、「1ファイル＝1クラス＝登場人物1人」と捉えてください。迷ったときは、「誰（どのクラス）が・何を持っていて（プロパティ）・何ができるか（メソッド）」の一覧表を紙に書き出すのがおすすめです。この表が自分で書けるなら、コードがどれだけ増えても迷子になりません。教材には各回の完成コードがファイルごとに載っているので、見取り図としても使えます。

Q7継承の書き方（maxHp などが入ってくるあたり）が、答えを見ても理解しきれない

一気に全部理解しようとしなくて大丈夫です。解答コードは1行ずつ分解して読むのがコツです。たとえば Enemy クラスなら、①extends Character（親の性質を引き継ぐ）→ ②constructor で super(name, hp)（親の初期化を呼ぶ）→ ③そのあとに自分だけのプロパティ（maxHp）を足す——という3段階で、それぞれ別のことをしています。

順番としては「写経（見ながら書く）→ そらで書く」の2周がおすすめです。分解してもなお引っかかる行があれば、「この行が分からない」とアンケートで聞いてください。その粒度の質問がいちばん答えやすいです。復習はLecture 5（継承・抽象クラス）へ。

Q8main.ts が全体として何をしているのか分からない

main.ts はプログラムの入口（司令塔）です。クラスのファイル（character.ts など）は「設計図の置き場」で、それだけでは何も起きません。main.ts が設計図を import して、new でインスタンス（実物）を作り、メソッドを呼んで動かします。実行したときに上から順に流れるのは main.ts だけ——「動きを追いたければ main.ts を上から読む」と覚えてください。
