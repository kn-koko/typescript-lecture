# 講義動画 → テキスト教材 パイプライン

オンデマンド講義（字幕付き動画）とソースリポジトリのコミット履歴から、**lecture単位のテキスト教材**を作る。各回は「講師の解説（字幕ベース）」＋「そのとき書いたコード／操作画面」で構成する。

## 入力
- `docs/subtitles/*.ja.vtt` … 各動画の字幕（YouTube自動字幕形式・ワード単位タイムスタンプ）
- `docs/video/*.mp4` … 各動画（`.gitignore` 済み。ファイル名は字幕とID対応）
- Gitのコミット履歴 … 授業の進行と課題の成果物

## 成果物
- 各lectureのHTML（Artifactツールで公開、または `docs/textbook/` にスタンドアロンHTML＋目次）
- 画像はJPEGを **data URIで埋め込み**、各HTMLを単体表示可能にする

---

## 手順

### 0. 全体設計（着手前にユーザーへ確認）
- **背骨**：動画番号順（推奨）／コミット順／ハイブリッド
- **操作系動画**（Git/GitHub等）：本文に章として含める／付録／除外
- **出力**：HTMLアーティファクト／リポジトリ静的サイト、lecture単位分割
- **課題コミット**（動画に対応しない演習）：各lecture末に「演習」章／別冊／省略
- 動画↔コミットは1:1でないことが多い。**字幕の内容から対応コミットを判断し、不明ならユーザーに聞く**。

### 1. 字幕を時刻付き文字起こしに変換
```bash
python3 scripts/vtt2txt.py "docs/subtitles/<file>.ja.vtt" > /tmp/<key>.txt
```
`[MM:SS] 本文` 形式。約2秒ごとの連続発話なので、つなげて全文として読む。これを解説文（ナレーション）の素にする。

### 2. コード系の回：gitから実コードを掲載
- `git show <commit>:<path>` で**コミットのコードを正確に取得**（推測しない）。コード用ファイルを `code<N>/name.ts` に保存。
- テンプレHTMLに `{{CODE:name}}` を置き、`build_lecture.py` の正規表現ハイライタで着色（画像化しない）。
- 課題コミット（Counter, Book, attack, 戦闘 等）は各lecture末の**「演習」章**に、問題文＋回答コード＋POINTで載せる（ナレーション無し）。問題文は回答コードから再構成し、冒頭ノートに「再構成」と明記。

### 3. 操作系の回：動画からスクショを自動抽出（QC必須）
1. 文字起こしを読み、**ショットリスト**（撮るべき画面＋時刻）を作る。
2. 抽出：`ffmpeg -nostdin -loglevel error -y -ss <秒> -i "<mp4>" -frames:v 1 out.png`（フル解像度）。QC用は `-vf scale=960:-2 -q:v 4` で軽量に。
3. **【必須】全フレームをReadツールで目視確認**。読み込み中・切替・空白カットを弾く。NGなら前後の時刻を数点サンプルして選び直す。※このQCを省くと取り違える（過去に発生）。
4. 確定フレームをフル解像度PNG→JPEG化：`ffmpeg -i x.png -q:v 3 x.jpg`（1920pxのまま。data URIサイズを抑える）。
5. テンプレに `{{IMG:<chap>/<nn>}}` を置く。

### 4. 使い捨て例の回（コードがリポジトリに無い）
- 画面に書いたコードをスクショで押さえるか、画面から正確に書き起こしてコードブロック化。冒頭ノートに「リポジトリ未収録・動画から書き起こし」と明記。

### 5. ビルド（コード注入＋画像埋め込み）
```bash
python3 scripts/build_lecture.py template.html out.html [codeDir]
```
- `{{CODE:name}}` → `codeDir/name.ts` を構文ハイライトして注入
- `{{IMG:chap/nn}}` → `docs/images/chap/nn.jpg` を data URI 化して注入
- 出力後 `unresolved=[]` を確認。

### 6. 出力
- **アーティファクト**：`artifact-design` スキルを読んでから Artifact ツールで公開。
- **リポジトリ静的サイト**：各HTMLを `<!doctype html><html><head>…</head><body>…</body></html>` でラップ（先頭 `<title>` を head へ、`</style>` で head/body 分割）して `docs/textbook/lectureN.html` に配置。目次 `index.html`（相対リンク）と、Pages `/docs` 配信用のルートリダイレクト `docs/index.html` を用意。`docs/images` は再生成可なので `.gitignore`。

## デザイン規約（全冊で統一）
- **書体**：ナレーション＝セリフ体（講師の声）、コード＝等幅、見出し＝サンス。声とコードを書体で対比。
- **配色**：TypeScriptブルー `#2f6fb3` 基調、青みニュートラル。ダーク/ライト両対応（`prefers-color-scheme` ＋ `:root[data-theme]`）。
- **構成**：目次（アンカー）→ 各章「解説→コード/手順→スクショ→POINT」。操作系は番号付きステップ。演習は各冊末。
- 冒頭に点線ノートで「その回の作り方（元コミット・推定/補完箇所）」を明記。

## 注意
- **コードは必ずgitから**（正確性）。**スクショは必ず全カット目視QC**（ユーザーの明示要望）。
- 動画の説明用の名前と実コードの名前がずれる場合は**実コードに統一**し、冒頭ノートで断る（例：説明"action"→実コード`attack`）。
